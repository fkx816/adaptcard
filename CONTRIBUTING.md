# Contributing to adaptcard

Thanks for your interest in contributing.

## How to contribute

- Open an issue for bug reports or feature proposals.
- For larger changes, discuss design first in an issue.
- Keep pull requests focused and small when possible.
- Add or update docs when behavior changes.

## Development principles

- Knowledge-point-first design
- Reproducible scheduling logic
- Privacy-by-default for learner data
- No secrets in code, tests, or examples

## Commit style

Use clear commit messages:

- `feat: ...`
- `fix: ...`
- `docs: ...`
- `chore: ...`

## Pull request checklist

- [ ] Code builds and tests pass locally
- [ ] Docs updated if needed
- [ ] No secrets added
- [ ] Backward compatibility considered
